# Wiki v3 — The Neural Memory Engine

> A generic, open-source memory orchestrator for agentic systems.
> Biologically-inspired. Crash-resistant. Privacy-first. Self-healing.
> Builds on Karpathy's LLM Wiki (v1) and rohitg00's LLM Wiki v2, then fixes what neither addresses: lifecycle, reconsolidation, governance, and operational stability.

---

## 0. Preamble

Most memory layers for LLM agents fail in the same three ways:

1. **They accumulate forever.** Every observation becomes a permanent fact. No decay, no supersession, no pruning. The knowledge base becomes a junk drawer within weeks.
2. **They crash under their own weight.** Multiple services on multiple ports, in-memory state lost on restart, no transactional boundaries, no circuit breakers. Operators learn to restart instead of fix.
3. **They leak.** PII, secrets, and private thoughts flow straight from ingestion into embeddings and cloud LLMs. Privacy is bolted on after the first incident.

Wiki v3 is designed to not do any of that. It takes one metaphor seriously — the human brain — and lets that metaphor constrain the design. Neurons fire, synapses strengthen, unused paths fade, contradictions trigger reconsolidation, sleep consolidates, and a blood-brain barrier keeps private things private.

The metaphor is not decoration. Each biological mechanism maps to a specific, testable protocol.

---

## 1. Design Philosophy

**Principle 1 — One process, many modules.**
A single long-running async process hosts every module. No inter-service RPCs for core memory operations. Scale-out is a later concern, not a starting assumption.

**Principle 2 — Append-only is the source of truth.**
Every input to the system is an immutable event in a log. All derived state (neurons, synapses, summaries, embeddings) can be rebuilt from the log. This makes replay, debugging, and disaster recovery trivial.

**Principle 3 — Tiers are explicit, not implicit.**
Working memory is not semantic memory is not procedural memory. Each tier has its own storage, retention, promotion rule, and purpose. "Everything is a node" is an anti-pattern.

**Principle 4 — Confidence, provenance, and scope are on every neuron.**
Not optional metadata. Not added later. Day-one schema.

**Principle 5 — Self-healing is declarative.**
Invariants are declared in config. The healer checks them, repairs what it can, quarantines what it can't, and always logs what it did. No silent corruption.

**Principle 6 — Privacy is a scope, not a filter.**
A neuron's scope (`private` / `shared` / `public`) determines where it lives, how it is encrypted, and who can retrieve it. The scope is checked on every read.

**Principle 7 — Forgetting is a feature.**
Unused synapses decay. Superseded neurons fade. Hard-delete ("protective forgetting") is supported with tombstones that prevent re-ingestion of the same content.

**Principle 8 — Biology is a guide, not a religion.**
When the metaphor stops helping, drop it. We use Hebbian learning because it maps cleanly to co-retrieval reinforcement. We do not simulate action potentials.

## 3. Biology → Engineering Mapping

This table is the contract. If a proposed change breaks one of these mappings, the design is wrong.

| Biological mechanism | Engineering construct | Purpose |
|---|---|---|
| Neuron | Row in `neurons` table | Atomic memory unit |
| Synapse | Row in `synapses` table | Typed, weighted connection between neurons |
| Dendrite | Ingestion pipeline | Receive signals from outside |
| Axon | Retrieval pipeline | Send signals outward |
| Hebbian learning | Synapse weight increases when both neurons co-retrieved | "Fire together, wire together" |
| LTP (long-term potentiation) | Repeated activation raises neuron activation score | Reinforcement |
| LTD (long-term depression) | Idle neurons lose activation over time | Decay |
| Synaptic pruning | Pruner removes synapses below threshold during consolidation | Keep graph sparse |
| Hippocampal consolidation | Promoter moves working → episodic → semantic | Offline compression |
| Sleep cycle | Scheduled consolidator run | Batch-time housekeeping |
| Reconsolidation | On recall, neuron enters labile state; updates apply in place | The unlearn/relearn primitive |
| Schema assimilation | New evidence strengthens or supersedes matching neurons | Integration |
| Protective forgetting | Hard-delete with tombstone | Trauma / GDPR equivalent |
| Blood-brain barrier | Privacy layer: scrub on ingress, redact on egress | Keep secrets inside, keep toxins out |
| Immune response | Contradiction detector | Flag, quarantine, or supersede bad data |
| Homeostasis | Healing layer with invariants | Keep the system in a healthy range |

## 4. Component Catalog

For each module: **purpose, inputs, outputs, data shapes, failure modes, connections, healing behavior.**

### 4.1 Sensory Layer (ingress)

**Purpose.** Receive raw input from the outside world and turn it into a single, normalized, classified, scrubbed `Event`. Nothing downstream ever sees raw input.

**Biological analog.** Sensory neurons + thalamic relay.

**Submodules.**
- **Adapters** — one per source type (HTTP, CLI, file watcher, webhook, message queue). Each adapter knows only its own protocol and yields a raw envelope.
- **Normalizer** — turns any raw envelope into the canonical `Event` shape.
- **Classifier** — assigns `type` (observation, correction, instruction, question, reflection), tentative `scope` (private/shared/public), and extracts candidate entities for later.
- **PII Scrubber** — regex pass + optional LLM double-check. Replaces matched substrings with typed tokens (`[REDACTED:email]`, `[REDACTED:otp]`, `[REDACTED:key]`) and stores the mapping in the vault if the scope is private and retrievable, or discards it if the scope is public.

**Input.** Arbitrary bytes + metadata from adapters.
**Output.** One `Event` appended to the log.

**Failure modes.**
- Malformed input → reject, log, do not append.
- Scrubber miss (detected later by healing) → quarantine event, re-scrub, replace.
- Duplicate input (by content hash) → dedupe, do not append.

**Connections.** Emits to Event Log only. Does not touch neurons.

**Healing behavior.** Scrubber ruleset is versioned. When rules update, a healing task re-scrubs all past events created under older rule versions.

---

### 4.2 Event Log (WAL)

**Purpose.** The only source of truth. Append-only, content-hashed, monotonically ordered, durable.

**Biological analog.** Short-term sensory memory / the iconic store.

**Schema.**
```
events(
  id              BIGSERIAL PRIMARY KEY,
  ts              TIMESTAMPTZ NOT NULL DEFAULT now(),
  type            TEXT NOT NULL,
  scope           TEXT NOT NULL,         -- private | shared | public
  source          TEXT NOT NULL,
  payload         JSONB NOT NULL,        -- already normalized, already scrubbed
  content_hash    BYTEA NOT NULL UNIQUE, -- dedup key
  scrubber_rev    INT NOT NULL,          -- which PII ruleset produced this
  parent_event_id BIGINT REFERENCES events(id)   -- for corrections/supersessions
)
```

**Rules.**
- Never updated, never deleted except by protective forgetting.
- All other tables are derivable from this one.
- Reindex, rebuild, or replay is a function of `events` only.

**Failure modes.**
- Disk full → reject new events, alert, degrade to read-only mode (no silent loss).
- Clock skew → use monotonic `id`, not `ts`, for ordering.

**Connections.** Written by Sensory. Read by Working Memory, Consolidator, Healer, Observer.

---

### 4.3 Working Memory

**Purpose.** Hold the recent event stream in-RAM for fast access by the current agent turn. Bounded size, rebuilt from the event log on startup.

**Biological analog.** Prefrontal working memory — small, fast, volatile.

**Shape.**
- Ring buffer, configurable size (default: last 1,000 events or last 60 minutes, whichever larger).
- Indexed by session key, entity, and scope.
- Rebuilt on process start by replaying the tail of the event log.

**Failure modes.**
- Process crash → buffer lost, automatically rebuilt on restart. No data loss because log is durable.
- Buffer overflow → oldest events evicted; still retrievable from semantic memory if they were promoted.

**Connections.** Reads from Event Log. Fed to Consolidator on every tick. Queried by Retrieval for "recent context".

---

### 4.4 Privacy Layer (Blood-Brain Barrier)

**Purpose.** Three jobs: scrub on ingress, redact on egress, encrypt the vault.

**Biological analog.** The actual blood-brain barrier plus the dissociation between declarative and private experience.

**Components.**
- **PII Patterns** (`config/privacy_patterns.yaml`) — regex catalogue: emails, phone numbers, card numbers, national IDs, OTPs near keyword, API keys by format (sk-*, ghp_*, AIza*, etc.), private URLs. User-extensible.
- **Scope Enforcer** — every retrieval call includes a `requestor_scope`. Neurons with higher scope than the requestor are not returned. Default deny.
- **Secrets Vault** — encrypted store (libsodium sealed box or age). Secrets are replaced in neurons by opaque references like `vault://abc123`. Retrieval only returns the reference; a separate authorized call unseals it.
- **Egress Redactor** — last pass before any retrieval result leaves the process. Removes residual PII by pattern and enforces scope.

**Scopes.**
- `public` — may be shared with any LLM, any consumer, any log.
- `shared` — may be returned to authenticated requestors in the same scope.
- `private` — stays local; only local models and the authorized human see it.

**Failure modes.**
- Scrubber miss → healer re-scrubs later; severity logged.
- Scope leak (private returned to shared requestor) → critical invariant violation; process refuses further retrievals until acknowledged.

**Connections.** Wraps both ingress (Sensory) and egress (Retrieval). Vault is a sibling storage.

### 4.5 Storage (the Cortex)

**Purpose.** Durable, queryable storage of neurons, synapses, episodes, patterns, and derived indexes.

**Default backend.** SQLite + sqlite-vec. One file. Zero ops. Suitable for single-machine deployments up to low millions of events.

**Scale-up backend.** Postgres + pgvector, selected by config flag. Same schema, same queries.

**Tables.**

```
neurons(
  id              BIGSERIAL PRIMARY KEY,
  kind            TEXT NOT NULL,        -- entity | fact | session_summary | pattern
  tier            TEXT NOT NULL,        -- working | episodic | semantic | procedural
  content         TEXT NOT NULL,        -- short, human-readable
  embedding       VECTOR(N),
  activation      REAL NOT NULL DEFAULT 0.5,  -- 0..1 current activation
  confidence      REAL NOT NULL DEFAULT 0.5,  -- 0..1 how trusted
  scope           TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL,
  last_fired_at   TIMESTAMPTZ NOT NULL,
  fire_count      INT NOT NULL DEFAULT 0,
  source_count    INT NOT NULL DEFAULT 1,     -- distinct events supporting it
  superseded_by   BIGINT REFERENCES neurons(id),
  superseded_at   TIMESTAMPTZ,
  provenance      JSONB NOT NULL              -- { event_ids: [...], extractor_rev: N }
)

synapses(
  id              BIGSERIAL PRIMARY KEY,
  pre_id          BIGINT NOT NULL REFERENCES neurons(id),
  post_id         BIGINT NOT NULL REFERENCES neurons(id),
  rel_type        TEXT NOT NULL,        -- uses | depends_on | contradicts | caused | fixed | supersedes | mentioned_with | ...
  weight          REAL NOT NULL DEFAULT 0.1,  -- 0..1 strength
  created_at      TIMESTAMPTZ NOT NULL,
  last_fired_at   TIMESTAMPTZ NOT NULL,
  fire_count      INT NOT NULL DEFAULT 0,
  provenance      JSONB NOT NULL,
  UNIQUE (pre_id, post_id, rel_type)
)

episodes(
  id              BIGSERIAL PRIMARY KEY,
  session_key     TEXT NOT NULL,
  started_at      TIMESTAMPTZ NOT NULL,
  ended_at        TIMESTAMPTZ NOT NULL,
  summary         TEXT NOT NULL,
  scope           TEXT NOT NULL,
  neuron_ids      BIGINT[] NOT NULL,    -- neurons involved / created
  event_ids       BIGINT[] NOT NULL
)

patterns(
  id              BIGSERIAL PRIMARY KEY,
  name            TEXT NOT NULL,
  description     TEXT NOT NULL,
  trigger         JSONB NOT NULL,       -- conditions that activate it
  steps           JSONB NOT NULL,
  support         INT NOT NULL,         -- how many episodes support it
  confidence      REAL NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL,
  last_fired_at   TIMESTAMPTZ NOT NULL
)

vault(
  ref             TEXT PRIMARY KEY,     -- "vault://..." reference
  ciphertext      BYTEA NOT NULL,
  algo            TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL
)

tombstones(
  id              BIGSERIAL PRIMARY KEY,
  content_hash    BYTEA NOT NULL UNIQUE,
  reason          TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL
)

healing_log(
  id              BIGSERIAL PRIMARY KEY,
  invariant       TEXT NOT NULL,
  detected_at     TIMESTAMPTZ NOT NULL,
  action          TEXT NOT NULL,        -- repaired | quarantined | skipped
  target          JSONB NOT NULL,
  result          JSONB NOT NULL
)
```

**Indexes.**
- FTS on `neurons.content` (BM25 via sqlite-fts5 or Postgres tsvector).
- Vector index on `neurons.embedding`.
- Adjacency index on `synapses(pre_id)` and `synapses(post_id)`.

**Failure modes.**
- Corruption → rebuild from `events`. All storage tables are derived.
- Drift (storage disagrees with log) → healer detects and rebuilds.

### 4.6 Consolidator (the Sleep Cycle)

**Purpose.** Run offline-style jobs that promote working memory to higher tiers, reinforce active neurons, decay idle ones, detect contradictions, and trigger reconsolidation.

**Biological analog.** Slow-wave sleep (hippocampal replay → cortical integration) plus REM (integration, creative association).

**Schedule.**
- **Light pass** — every 60 seconds. Cheap work only: working→episodic, decay tick, metrics.
- **Deep pass** — every N minutes (default 15). Expensive work: episodic→semantic extraction, contradiction detection, pattern induction, synaptic pruning.

**Sub-routines.**

1. **Promoter.** Reads recent working-memory events. When a session closes (idle timeout or explicit end), creates an episode with an LLM-generated summary, extracted entities (neurons), and their relationships (synapses). Does not duplicate existing neurons — looks up by content hash + embedding similarity and either reinforces or creates.
2. **Reinforcer (LTP).** For every neuron retrieved or co-retrieved in the last window, bumps activation and fire_count. For every synapse traversed, bumps weight.
3. **Decayer (LTD).** Exponential decay on activation and synapse weight per idle tick. Rate differs by tier — procedural decays slowest, working decays fastest.
4. **Pruner.** Deletes synapses with weight below threshold and fire_count below threshold. Marks unreferenced neurons as orphan candidates (healer decides).
5. **Contradiction Detector.** For each new neuron, finds semantically similar existing neurons via vector search. If content disagrees (LLM-judged), emits a `contradiction` event for the healer.
6. **Pattern Inducer.** When the same sub-graph (same relationship shape, same entity kinds) appears in M+ episodes, promotes it to a `pattern` row.

**Failure modes.**
- LLM timeout during promotion → episode stays in working memory, retried next tick, no partial writes.
- Pruner over-eager → restore from `events` replay.

**Connections.** Reads Event Log + Storage. Writes Storage. Emits events to Healer on anomalies.

---

### 4.7 Retrieval (Associative Recall)

**Purpose.** Given a query and a requestor scope, return the most relevant neurons + context.

**Biological analog.** Cue-driven associative recall.

**Pipeline.**
1. **Query preprocessing** — normalize, extract entities, embed.
2. **Three retrieval streams in parallel:**
   - **BM25** over `neurons.content`.
   - **Vector search** over `neurons.embedding`.
   - **Graph walk** starting from entities named in the query, breadth-limited by synapse weight.
3. **Reciprocal Rank Fusion** combines the three rankings.
4. **Reconsolidation tick** — top-K returned neurons have their `last_fired_at` and `fire_count` updated; their synapses likewise. This is the *only* place where retrieval has a side effect.
5. **Privacy filter** — drop any neuron with scope exceeding requestor scope.
6. **Redactor** — final egress pass; replace vault references with opaque tokens unless the call was explicitly `unseal=true` with authorization.
7. **Package** — return `{neurons, synapses, episodes, citations}`.

**Failure modes.**
- One stream fails (e.g. vector DB down) → degrade gracefully, use remaining streams, flag in response metadata.
- Embedder timeout → fall back to BM25 + graph only.

**Connections.** Reads Storage. Updates Storage (reconsolidation side effect). Calls Privacy layer on egress.

### 4.8 Healing Layer

**Purpose.** Keep the system in a healthy range by declaring invariants and running detect→diagnose→repair→verify→learn cycles.

**Biological analog.** Homeostasis + immune response.

**Invariants (declared in `config/invariants.yaml`).** Each invariant has a name, a SQL or Python predicate, a severity, and a repair action. Examples:

```yaml
- name: no_orphan_synapses
  check: "SELECT count(*) FROM synapses s WHERE NOT EXISTS (SELECT 1 FROM neurons n WHERE n.id = s.pre_id) OR NOT EXISTS (SELECT 1 FROM neurons n WHERE n.id = s.post_id)"
  expect: 0
  severity: high
  repair: delete_orphan_synapses

- name: no_uncleaned_events
  check: "SELECT count(*) FROM events WHERE scrubber_rev < :current_rev"
  expect: 0
  severity: medium
  repair: rescrub_events

- name: no_contradictory_active_neurons
  check: "SELECT count(*) FROM synapses WHERE rel_type = 'contradicts' AND weight > 0.5 AND pre_id NOT IN (SELECT id FROM neurons WHERE superseded_by IS NOT NULL) AND post_id NOT IN (SELECT id FROM neurons WHERE superseded_by IS NOT NULL)"
  expect: 0
  severity: high
  repair: resolve_contradiction

- name: scope_leak_impossible
  check: (python)  # compares last 1000 retrievals' requestor_scope vs returned_scope
  expect: 0
  severity: critical
  repair: halt_and_alert
```

**Repair actions library.**
- `delete_orphan_synapses` — removes synapses whose endpoints no longer exist.
- `rescrub_events` — reruns current PII ruleset on old events, supersedes neurons derived from them.
- `resolve_contradiction` — calls LLM judge with both sides, supersedes the weaker claim, records reasoning.
- `merge_duplicates` — for near-identical neurons, merges into the higher-confidence one.
- `reembed_stale` — when the embedder version changes, re-embeds in batches.
- `rebuild_index` — rebuild FTS/vector/graph indexes from `neurons` and `synapses`.
- `full_replay` — nuclear option. Rebuild all derived state from `events` only.
- `halt_and_alert` — flip system to read-only, emit alert. Used for invariants whose violation implies a security incident.

**Quarantine.** Things the healer cannot auto-repair go to a quarantine table with a human-readable explanation. They stop participating in retrieval and consolidation until a human acknowledges.

**Learn step.** After each repair, the healer writes a `healing_log` entry. If the same invariant fires N times in a window, the system auto-proposes a config change (e.g. tighter PII pattern, lower pruner threshold) and files it as a pending change. Humans approve.

**Failure modes.**
- Repair itself fails → quarantine, alert, do not retry blindly.
- Invariant is poorly specified → healer has a "dry-run" mode for new invariants. Reports what it *would* have done, without doing it. Human reviews before enabling.

**Connections.** Reads everything. Writes anything. Cannot be disabled, only tuned.

### 4.9 Observer

**Purpose.** Metrics, structured logs, replay tool.

**Exports.**
- Prometheus-compatible metrics at `/metrics` (counts, latencies, invariant violations).
- Structured JSON logs.
- `replay` CLI: given a point in time, rebuild any derived table from `events` and diff against current state — a powerful debugging tool.

Never on the hot path of retrieval.

## 5. Data Flows

### 5.1 Ingest flow

```
raw input
   │
   ▼
adapter.receive()
   │
   ▼
normalizer.canonicalize()  ──► { type, scope_hint, source, payload }
   │
   ▼
classifier.classify()      ──► { type, scope, candidate_entities }
   │
   ▼
scrubber.scrub(scope)      ──► payload with [REDACTED:*] tokens + vault refs
   │
   ▼
dedupe by content_hash     ──► drop if seen, tombstoned, or matches superseded
   │
   ▼
events.append(event)
   │
   ▼
working_memory.ingest(event)
   │
   ▼
(consolidator picks up on next tick)
```

Note: ingest completes in <100ms on the hot path. Everything that is not time-critical (embedding, LLM classification, KG extraction) happens in the consolidator.

### 5.2 Retrieval flow

```
request(query, requestor_scope)
   │
   ▼
preprocess(query)          ──► { normalized, entities, embedding }
   │
   ├──► bm25.search()       ──┐
   ├──► vector.search()     ──┼──► rrf.fuse()  ──► top-K neurons
   └──► graph.walk()        ──┘
   │
   ▼
reconsolidation.tick(top-K)  — bump activation & synapse weights
   │
   ▼
privacy.filter(scope)
   │
   ▼
redactor.finalize()
   │
   ▼
response { neurons, synapses, episodes, citations, degraded_streams? }
```

### 5.3 Consolidation flow (one deep pass)

```
for each session closed in window:
    events = read(session_events)
    summary, entities, relations = llm.extract(events)

    for e in entities:
        n = find_or_create_neuron(e)    # dedupe by hash + embedding similarity
        reinforce(n) OR create(n)

    for r in relations:
        s = find_or_create_synapse(r)
        reinforce(s) OR create(s)

    episodes.insert(...)

for each neuron not fired in [window]:
    decay(n)

for each synapse not fired in [window]:
    decay(s)
    if weight < prune_threshold: delete(s)

for each new neuron:
    similar = vector.search(n.embedding)
    for candidate in similar:
        if llm.contradicts(n, candidate):
            emit_event(contradiction, {a: n, b: candidate})

induce_patterns()
emit_metrics()
```

### 5.4 Healing flow

```
every minute:
    for inv in invariants:
        if not inv.holds():
            log(inv.name, severity)
            if auto_repair_enabled(inv):
                result = inv.repair()
                verify(inv)
                if still broken: quarantine()
            else:
                quarantine()
        else:
            log_ok(inv.name)

    if same invariant fires > N times in window:
        propose_config_change()
```

### 5.5 Unlearn/Relearn (Reconsolidation Protocol)

This is the heart of "brain-like memory." Spelled out fully:

```
when neuron N is retrieved:
    N.state = labile
    N.labile_since = now

while N.state == labile and (now - N.labile_since) < RECONSOLIDATION_WINDOW (default 5m):
    if an event arrives that provides updated content for N:
        if event supports N:
            N.confidence += α
            N.source_count += 1
            N.activation += β        # LTP
        elif event contradicts N:
            new = create_neuron(event.content)
            N.superseded_by = new.id
            N.superseded_at = now
            # old N is not deleted — still readable with include_superseded=true
            emit_event(supersession, {old: N, new: new, reason: ...})
        elif event refines N:
            merged = llm.merge(N.content, event.content)
            N.content = merged
            N.provenance.event_ids += event.id
            # in-place update is allowed ONLY inside the reconsolidation window

when window closes:
    N.state = stable
    write N
    if N was not reinforced during window:
        N.activation *= decay_factor  # not firing during labile = mild LTD
```

**Why this matters.** In classical systems, "unlearn" means delete. In a neuron-like system, unlearn means supersede with full audit, and relearn means new evidence can re-elevate the superseded version (creating a new neuron that cites both histories). The consolidator on its next deep pass inspects superseded neurons with recent supporting evidence and promotes a merged successor if warranted.

**Hard-delete ("protective forgetting")** is a separate path, triggered explicitly (GDPR, security incident, user request). It:
1. Tombstones the `content_hash` so the same content cannot re-enter.
2. Deletes the neuron and all its synapses.
3. Deletes the originating events.
4. Re-runs affected invariants.
5. Logs the deletion (not the content) to `healing_log` with reason.

## 6. Privacy Model

### 6.1 Scope Lattice

```
public  ≤  shared  ≤  private
```

Every neuron has exactly one scope. A retrieval requestor presents its own scope. The enforcer returns only neurons with `neuron.scope ≤ requestor.scope`.

### 6.2 Scope Assignment at Ingest

The classifier assigns scope using, in order:
1. Explicit scope on the incoming event (caller-declared, honored).
2. Source defaults (e.g. SMS from a telco-identified sender → `private`; public web fetch → `public`).
3. Content signals (presence of PII patterns → `private` even if source said otherwise; auto-promotion only, never auto-demotion).

Scope can only be raised later (private→...no, private is the ceiling), not lowered. Demoting a neuron requires an explicit re-classification event.

### 6.3 Secrets Vault

Any content that matches a secret pattern (API keys, passwords, OTPs, tokens) is extracted from the neuron content, encrypted, and stored in the vault. The neuron keeps only an opaque `vault://...` reference.

Vault unseal is a separate, authenticated call. Retrieval by default never unseals. This means embeddings never contain secrets.

### 6.4 Egress Redactor

Final pass on retrieval output:
- Replace vault references with tokens unless authorized unseal.
- Re-run current PII patterns (belt and braces) — if a pattern matches, drop the neuron from the response and emit a healing event.
- Strip provenance event IDs if requestor scope is below private.

### 6.5 "Like a Human"

- Humans don't broadcast internal thoughts → scope defaults are conservative.
- Humans forget trauma deliberately → protective forgetting with tombstone.
- Humans tell different things to different people → scope enforcement on retrieval.
- Humans don't remember passwords verbatim; they remember that a password exists → vault references, not raw values.

## 7. Crash & Conflict Elimination

This section enumerates the failure classes that Digital Brain (and most memory systems) hit, and the specific mechanism in Wiki v3 that prevents each.

| Failure class | Wiki v3 mechanism |
|---|---|
| In-memory state lost on restart | All durable state in event log + storage; working memory rebuilt from log on boot |
| Service A up, service B down | No inter-service calls on hot path; one process |
| Race condition on write | All writes via storage layer with row-level locking; idempotency via content hash |
| Double-write on retry | Dedupe by content hash + event-level idempotency keys |
| Cascading failure from LLM timeout | Circuit breaker on every LLM call; consolidator falls back to non-LLM mode and retries |
| Silent corruption | Invariants checked every minute; healer repairs or quarantines |
| Embedder upgrade breaks search | Embeddings tagged with `embedder_rev`; healer reembeds on mismatch |
| Schema migration breaks old rows | Migrations are additive-only; removals only via soft-delete + backfill |
| Clock skew | Ordering uses monotonic `events.id`, not `ts` |
| Runaway resource use | Each background job has a deadline; consolidator yields cooperatively |
| Poison message | Events that cause repeated consolidator failure are quarantined after 3 tries |
| Storage corruption | Full replay from event log as nuclear repair |
| PII leak downstream | Scope enforcer + egress redactor; leaked neuron traceable via provenance |
| Deadlock | Single writer for each table; readers non-blocking |
| Split brain (multi-instance) | MVP is single-instance; multi-instance uses leader election, deferred to v1.1 |

POST   /v1/heal/run               — manual healer trigger
GET    /v1/heal/report            — invariant status report

GET    /ready                     — readiness (DB reachable, log writable)

### 9.2 CLI

```
wiki-v3 init                      — create DB, config, keypair
wiki-v3 ingest <file|-|url>       — one-shot ingest
wiki-v3 recall "<query>" [--scope]
wiki-v3 doctor                    — full invariant report
wiki-v3 replay --to <event_id>    — rebuild derived state up to event_id
wiki-v3 vault unseal <ref>        — authenticated
wiki-v3 forget --content-hash <h> — protective delete
```

### 9.3 Contract Rules

- Every request carries an idempotency key; duplicates are no-ops.
- Every response includes `{request_id, warnings[], degraded: bool}`.
- Errors are typed; never bare stack traces to clients.

## 12. Testing Strategy

- **Unit tests** per module, no external deps, mockable LLM client.
- **Property tests** (Hypothesis) for invariants: "for any event stream, after full consolidation, all invariants hold."
- **Golden tests** on prompts with pinned model outputs.
- **Chaos tests**: kill the process mid-consolidation, verify recovery.
- **Scope-leak tests**: specifically try to exfiltrate `private` via `shared` requestor; assert 0 leaks.
- **Reconsolidation tests**: assert unlearn/relearn preserves audit trail.

No real data in tests. All fixtures are synthesized by a `faker`-style generator.

## 13. Non-Goals (deliberately out of scope for v0.1)

- Multi-tenant server. One tenant per instance. Multi-tenancy is a v1.x concern.
- Multi-instance mesh sync. Single-writer only.
- Built-in UI. Ship JSON + CLI; let others build UIs.
- Specific LLM vendor. `llm/client.py` is provider-agnostic.
- Replacing an agent framework. Wiki v3 is a memory layer, not an agent.
- Training custom models. We consume embeddings and LLMs; we do not train them.

Healer       ── reads  →  Storage, Event Log

Observer     ── reads  →  everything

## 16. Governance (what must not change without a migration)

These are the contracts. Breaking any of them requires a major-version bump and a migration plan.

1. **Events are immutable.** Once appended, an event is never modified. Corrections are new events with `parent_event_id`.
2. **Derived state is disposable.** Neurons, synapses, episodes, patterns can always be rebuilt from events.
3. **Scope can be raised, never lowered automatically.**
4. **Secrets never appear in embeddings.** Vault references only.
5. **Invariants are declarative.** Code cannot silently bypass them.
6. **Provenance on everything.** Every neuron traces back to event IDs.
7. **Retrieval has exactly one side effect.** Reconsolidation tick. Nothing else.
8. **Single writer per table.** No multi-threaded writes to the same table.

## 17. How to use this document

This is the blueprint. To build Wiki v3 from it:

1. Start with **§8 (repo layout)** and **§11 (phase 0)**. Get a skeleton running with just `/health` on day one.
2. Keep **§3 (biology mapping)** and **§5 (data flows)** open in another tab as you code. They are the sanity checks.
3. Treat **§16 (governance)** as a test suite. Write property tests that assert each rule.
4. Before shipping any change, re-run **§12 (testing)** and `wiki-v3 doctor`.
5. When in doubt about whether to add a service, reread **§1 Principle 1**.

The goal is not to build everything in this document. The goal is to build the smallest system that satisfies the governance rules and can grow without needing a rewrite. Phase 0 is enough to be useful. Everything after that is compound interest.
